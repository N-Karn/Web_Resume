Featured case study

Protecting school funding by fixing broken reporting data.

A real example of how I like to work: start with a vague ‚Äúsomething is wrong with the numbers,‚Äù trace the data end-to-end, and ship a fix that protects both funding and trust.

Scenario

Catching a reporting defect before it cost an LEA real money

Problem One LEA flagged that their state report counts didn‚Äôt match internal expectations. If submitted as-is, the bad data could have led to a loss of funding tied to student membership/attendance. The issue wasn‚Äôt showing up as a single obvious ‚Äúbug‚Äù ‚Äì it was a mix of edge cases, historical data, and unclear rules.
Constraints The reporting window was already open, the LEA needed clarity quickly, and other customers were using the same logic. Any fix had to be accurate, low-risk, and well-documented so support and engineering could rely on it going forward.
What I did
  • Met with the LEA to clarify where the counts looked wrong and what ‚Äúcorrect‚Äù should look like in their context.
  • Traced the data from the source system through to the state reporting payloads, using SQL and API samples to reproduce the issue.
  • Performed root cause analysis to separate true logic defects from dirty or incomplete historical data.
  • Documented clear requirements and business rules for how these students should be counted, including edge cases.
  • Partnered with engineering to clean up existing data via API-safe updates and to adjust the reporting logic going forward.
Outcome

Immediate impact: The LEA was able to submit accurate data for the current cycle, avoiding a potential funding loss tied to under-reported students.

Longer term: The clarified rules, examples, and acceptance criteria were reused for other customers and future releases, making it easier for support, QA, and engineering to spot similar issues before they hit production.

Skills in play: requirements gathering, stakeholder communication, root cause analysis, SQL + API investigation, and coordinating data cleanup without interrupting day-to-day operations.
